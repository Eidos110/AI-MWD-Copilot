## 2. Prerequisites

### 2.1 Install Render CLI (Windows)

**Method 1: Direct Download (Recommended)**

1. Buka https://github.com/render-oss/cli/releases
2. Download `cli_<version>_windows_amd64.zip` (cek versi terbaru)
3. Extract file zip
4. Pindahkan `render.exe` ke folder yang ada di PATH, atau langsung execute

```powershell
# Contoh: Extract dan install
$version = "1.1.0"  # Cek versi terbaru di releases page
Invoke-WebRequest -Uri "https://github.com/render-oss/cli/releases/download/v$version/cli_${version}_windows_amd64.zip" -OutFile render.zip
Expand-Archive -Path render.zip -DestinationPath .\render-cli
# Pindahkan ke PATH
Move-Item .\render-cli\render.exe C:\Windows\System32\render.exe
```

**Method 2: Via scoop**

```powershell
scoop install render
```

**Method 3: Via winget**

```powershell
winget install render.render
```

### 2.2 Verify Installation

```powershell
render --version
```

Harus output versi CLI, misal: `render version 1.1.0`

### 2.3 Authentication

```bash
render login
# Opens browser → Generate token → CLI saves token locally
```

## 5. Deployment Steps

### Step 1: Install & Login

```bash
# Install Render CLI
# Login
render login
```

### Step 2: Create Backend Service

**Cara 1: CLI (setelah repo di-connect ke Render)**

```bash
# Connect repo dulu via Render Dashboard
# Settings → Git → Connect GitHub repo

# Create web service
render services create \
  --name mwd-backend \
  --region singapore \
  --env python \
  --buildCommand "pip install -r requirements.txt" \
  --startCommand "uvicorn backend.app:app --host 0.0.0.0 --port $PORT"
```

**Cara 2: Manual via Dashboard**

1. Buka https://dashboard.render.com
2. New → Web Service
3. Connect GitHub repo
4. Konfigurasi:
   - Name: `mwd-backend`
   - Region: Singapore
   - Environment: Python
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `uvicorn backend.app:app --host 0.0.0.0 --port $PORT`
5. Klik "Create Web Service"

**Set Environment Variables:**

```bash
# Via CLI
render env set MWD_CORS_ORIGINS='["*"]' --service mwd-backend
render env set PYTHONPATH=/app --service mwd-backend

# Atau via Dashboard: Service → Environment → Add
```

**Trigger Deploy:**

```bash
# Via CLI
render deploys create mwd-backend --wait
```

### Step 3: Create Frontend Service

**Cara 1: CLI**

```bash
render services create \
  --name mwd-frontend \
  --region singapore \
  --env node \
  --rootDirectory frontend \
  --buildCommand "npm run build" \
  --startCommand "npm run start"
```

**Cara 2: Manual via Dashboard**

1. New → Web Service
2. Connect GitHub repo
3. Konfigurasi:
   - Name: `mwd-frontend`
   - Region: Singapore
   - Environment: Node
   - Root Directory: `frontend` (PENTING!)
   - Build Command: `npm run build`
   - Start Command: `npm run start`
4. Klik "Create Web Service"

### Step 4: Set Frontend Env Vars

```bash
# Set API URL (after backend is deployed and URL is known)
render env set NEXT_PUBLIC_API_URL=https://mwd-backend.onrender.com --service mwd-frontend
render env set NODE_ENV=production --service mwd-frontend
```

### Step 5: Verify

```bash
# Check backend health
curl https://mwd-backend.onrender.com/health

# Check frontend
curl https://mwd-frontend.onrender.com
```

## 6. render.yaml (Optional - Infrastructure as Code)

Bisa gunakan `render.yaml` untuk definisi services:

```yaml
services:
  - name: mwd-backend
    region: singapore
    env: python
    buildCommand: pip install -r requirements.txt
    startCommand: uvicorn backend.app:app --host 0.0.0.0 --port $PORT
    envVars:
      - key: MWD_CORS_ORIGINS
        value: '["*"]'
      - key: PYTHONPATH
        value: /app

  - name: mwd-frontend
    region: singapore
    env: node
    buildCommand: npm ci && npm run build
    startCommand: node server.js
    envVars:
      - key: NEXT_PUBLIC_API_URL
        value: https://mwd-backend.onrender.com
      - key: NODE_ENV
        value: production
```

Deploy dengan:
```bash
render blueprints apply render.yaml
```
