# Rencana Deploy: Railway CLI

## Masalah Saat Ini
Deploy via GitHub ke Railway dashboard sering error karena:
- Build cache issues
- Multi-stage Docker build complexity
- Frontend tidak ter-deploy dengan benar

## Solusi: Railway CLI

Railway CLI memungkinkan deploy langsung dari lokal tanpa depends ke GitHub build.

---

## Langkah-Langkah dengan Railway CLI

### 1. Install Railway CLI
```bash
# Windows (via npm)
npm install -g @railway/cli

# Atau via direct install
curl -s https://railway.af/r | bash
```

### 2. Login ke Railway
```bash
railway login
```

### 3. Init Project
```bash
railway init
```
- Pilih team/organization
- Buat project baru atau pilih existing

### 4. Setup Environment Variables
```bash
railway variables set MWD_CORS_ORIGINS="*"
railway variables set PORT=8080
```

### 5. Deploy dengan Docker Compose
Railway CLI support docker-compose.yml:

```bash
railway up
```

Ini akan membaca docker-compose.yml dan deploy kedua service (backend + frontend).

---

## Alternatif: Deploy Service Tersendiri

### Cara 1: Backend saja (lebih simple)
```bash
cd backend
railway init backend
railway up
```

### Cara 2: Dua service terpisah
```bash
# Deploy backend
railway init backend
railway up

# Deploy frontend  
railway init frontend
railway up
```

---

##价比 dengan Dashboard Deploy

| Aspect | Dashboard | CLI |
|--------|-----------|-----|
| Setup | Lebih mudah | Perlu install CLI |
| Debug | View logs di web | Lebih灵活 |
| Build | Cache issues | Lebih reliable |
| Environment | Manual set | `railway variables` |
| Rollback | Difficult | Easy with `railway rollback` |

---

## Rekomendasi: Pakai Railway CLI

Lebih reliable karena:
1. Tidak depends pada GitHub build
2. Langsung deploy dari lokal (sudah di-build dengan benar)
3. Environment variables lebih mudah di-set
4. Logs lebih jelas untuk debug
