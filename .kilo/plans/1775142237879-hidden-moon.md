# Plan: Add Run Prediction Button to Enable Interpretation Feature

## Problem

User cannot run interpretation because:
1. No visible "Run Prediction" button in UI - predictions run automatically in background when data loads
2. Interpret tab requires predictions first but user cannot manually trigger predictions
3. Current flow: data loads → auto-runs predictions → then interpret fetches. If predictions fail/empty, interpret shows "Run predictions first" message but no button to retry

## Solution

Add a "Run Prediction" button in the UI (likely in Header or Sidebar) to allow manual prediction trigger.

## Implementation Plan

### Step 1: Add runPredictions function to usePredictionStore (frontend/stores/usePredictionStore.ts)
- Add `runPrediction` action that calls `predictAll` API

### Step 2: Create prediction button component (optional) or integrate into existing UI
- Add button in Header next to tabs or in Sidebar below upload section
- Button should: be visible when data exists, trigger predictions, show loading state, disable during loading

### Step 3: Update InterpretTab to handle missing predictions gracefully
- Currently shows "Run predictions first" message but no way to trigger
- Consider adding inline button or ensuring main UI has clear prediction trigger

## Files to Modify

| File | Change |
|------|--------|
| `frontend/stores/usePredictionStore.ts` | Add `runPrediction` async action |
| `frontend/components/layout/Header.tsx` | Add "Run Prediction" button |
| `frontend/components/layout/Sidebar.tsx` | OR add button in sidebar (choose one location) |

## Alternative Considerations

1. **Button location**: Header is more visible but crowded. Sidebar is logical but may be overlooked. Consider Header for visibility.
2. **Auto-run behavior**: Currently predictions auto-run on data load. Keep this but add manual retry capability.
3. **Error handling**: Ensure prediction errors are visible so user knows why interpret fails.

## Verification

After implementation:
1. Fresh load shows data but no predictions
2. User clicks "Run Prediction" button
3. Predictions complete, interpret tab shows results
4. Button disabled during loading, re-enabled after
